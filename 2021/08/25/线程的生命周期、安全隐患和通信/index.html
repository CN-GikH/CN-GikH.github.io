线程的生命周期、安全隐患和通信

发表于 2021-08-25 分类于 Java 阅读次数： Valine：
学习多线程的生命周期的控制，以及各种安全隐患的预防，线程之间的相互通信

一、线程的生命周期

  • 通过API提供的方法让线程声明周期的某个阶段，达到可控状态

1.sleep()方法

  • 在指定的毫秒数内让正在执行的线程休眠

  • 使当前正在执行的线程停留（暂停执行）指定的毫秒数，这取决于系统定时器和调度程序的精度和准确性

2.join()方法

方法重载
  • 等待调用该方法的线程结束后才能执行
    • 谁调用该方法，谁先执行
  • 等待调用该方法的线程，最多 time秒，然后才执行

3.线程的优先级

  • Fileds常量

    • Java为线程类提供了10个优先级
    • 优先级可以用整数1—10表示，超过范围会抛出异常
      • 最先等级 1
      • 默认等级 5
      • 最大等级 10

  • getPriority() 获取线程的优先级

  • setPriority(int b) 设置线程的优先级

  • 查阅main方法的优先级

实验：线程优先级的效果

二、线程的安全隐患

  • 隐患：存在发生的可能性,但不一定会发生
    • 多个线程是通过竞争CPU的时间获得运行机会
    • 多个线程什么时候获得CPU时间，占用多少
    • 一个正在运行着的线程在什么地方暂停，也是不确定的

1.线程同步

  • 只允许一个线程执行
  • synchronized同步锁
Ⅰ.同步代码块
  • 需要传递一个参数

  • 细节问题
Ⅱ.同步方法

  • 不想加静态就用this
  • 定义一个方法专门放synchronized

2.死锁（同步代码块嵌套）

  • 程序没有停止，而且线程不在执行

    • 当一个线程拥有A对象锁的时候，并且在等待B对象锁的时候，另一个线程拥有了B对象的锁，并在等待A锁
    • 这种情况就造成了死锁

  • 不要嵌套！！！！！！！！！！！！！！！！！！！

    • synchronized锁住
    • 必须用同一把锁

  • A线程执行之后，需要通知B执行，找一个中间者作为记录

2.三个进程间通信案例

  • while循环实现重复输出
  • 实现同步代码块，确保一个线程执行时不会被抢走执行权
    • synchronized 加同步锁
  • 三个线程要用一个锁对象
    • public static Object obj = new Object();
    • 因为main方法是静态的，所以锁对象也要静态
    • 锁对象很灵活，用什么都可以
  • 需要有记录员，记录线程之间通信的情况
    • 定义一个变量，记录是谁在发消息
      • int a = 1;
  • 线程1
    • 当a==1时，发送消息；并将a=2；唤醒所有线程
    • 当a!=1时，处于等待状态
  • 线程2
    • 当a==2时，发送消息；并将a=3；唤醒其他所有线程
    • 当a!=2时，处于等待状态
  • 线程3
    • 当a==3时，发送消息；并将a=1；唤醒其他所有线程
    • 当a!=3时，处于等待状态

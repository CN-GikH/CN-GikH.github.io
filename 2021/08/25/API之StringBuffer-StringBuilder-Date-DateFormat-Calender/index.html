API之StringBuffer\StringBuilder\Date\DateFormat\Calender

发表于 2021-08-25 分类于 Java
本文字数： 1.6k字     阅读时长 : 3 分钟
学习API的使用方法，认识并学习StringBuffer类、StringBuilder类、Date类、DateFormat类、Calender类

一、StringBuffer

1.什么是StringBuffer?

  • 线程安全
  • 可变的字符序列
    • 任意的改变长度
    • 任意改变内容

2.StringBuffer的内容

  • 包

    • 查看所在包

      • java.lang.StringBuffer
        • 在lang包下，不需要导包就可以使用

  • 修饰符：final 不能被无法继承

  • 构造方法

  • 成员方法

3.StringBuffer中的构造方法

Ⅰ.无参构造

  • StringBuffer()

    • 构造一个没有字符的字符串缓冲区，初始容量为16个字符。
    • 超出长度，自动扩容
Ⅱ. 带参一：StringBuffer(int capacity)

  • ```java
    public StringBuffer(int capacity)

    • 构造一个初始化为指定字符串内容的字符串缓冲区。
    • 手动给出默认值，系统给出长度

4.StringBuffer中的成员方法

Ⅰ.append()

  • ```java
    public StringBuffer append (数据类型 变量名)

    • 容量是新插入字符可用的存储量，超过此值将进行分配。
Ⅲ.查找序列

Ⅳ.删除序列

Ⅴ.插入数据到序列

  • offset - 偏移量
Ⅵ.其他方法
  • charAt方法同样可以使用，返回指定索引的char值
  • indexOf方法同样可以使用，查找指定字符串第一次出现的索引
  • length()返回长度
  • toString()返回此顺序中的数据的字符串
  • subString(start , end) 返回一个新的序列，当前的子序列

二、StringBuilder

1.步骤

  • 第一步：打开API，索引搜索StringBuilder类

  • 第二步：查看该类所在的包

    • lang包下不需要导包，直接使用

  • 第三步：查看概述部分

    • 一个StringBuilder的主要操作是append和insert方法，它们是重载的，以便接受任何类型的数据。
    • append方法总是在构建器的末尾添加这些字符; insert方法将字符添加到指定点。
    • StringBuilder实例不能安全使用多线程。 如果需要同步，那么建议使用[StringBuffer]
    • 修饰符：final
      • 不能被继承

  • 第四步：查看构造方法

  • 第五步：查看成员方法（功能）

2.构造方法

  • 与StringBuffer基本相同

3.成员方法

  • 与StringBuffer基本相同
  • append（）追加字符串到序列
  • insert（int offset , 数据类型 变量名）插入字符串到序列中
  • capacity() 返回容量值
  • 其他一些方法也可以使用，具体看API中解释

三、Date

1.步骤

  • 查看API，索引查找Date类

  • 查看包

    • 在Java.util包下

  • 查看修饰符

    • public 公共，可以直接使用

  • 查看方法概述

    • Date类代表了一个特定的时间，以毫秒的精度。

  • 查看构造方法

  • 查看成员方法

2.构造方法

Ⅰ.无参构造Date()

  • 分配一个 Date对象并对其进行初始化，以便它表示分配的时间，以最接近的毫秒为单位。

    • 获取当前时间
2.带参构造Date(long date)

  • 分配一个 Date对象，并将其初始化为表示从标准基准时间（称为“时代”）即1970年1月1日00:00:00 GMT起的指定毫秒数。

    • 获取指定时间的日期

3.成员方法

Ⅰ.after(Date when)

  • 测试此日期是否在指定日期之后

Ⅱ.before(Date when)

  • 测试此日期是否在指定日期之前。

Ⅲ.compareTo

  • 比较两个日期进行订购。

  • 如果参数Date等于此Date，值为0

  • 如果此日期在Date参数之前，该值小于0

Ⅳ.equals(Object obj)

  • 比较两个日期来平等。

  • 仅当long值相同时，两个Date对象相等

四、DateFormat

1.步骤

  • 看包

    • 需要导包

  • 修饰符

    • public
    • abstract抽象类
      • 不能直接实例化，要通过子类实例化

  • 方法概述

    • DateFormat是一个用于日期/时间格式化子类的抽象类，它以语言无关的方式格式化和分析日期或时间。
    • 就是时间和格式的转换
    • 真正实际上的时间是表示的毫秒值，Long类型

  • 直接子类SimpleDateFormat，可以拿来使用

2.构造方法

Ⅰ.无参构造

  • 创建一个新的日期格式。

3.成员方法

如果变量名打印出来是一个地址值，说明没有重写toString方法

4.子类SimpleDateFormat自定义时间格式

Ⅰ.无参构造创建对象

  • 构造一个 SimpleDateFormat使用默认模式和日期格式符号为默认的 [FORMAT]区域设置。

Ⅱ.有参构造创建对象

  • 使用给定的模式和日期格式符号构造一个 SimpleDateFormat 。

  • 创建一个时间规则

Ⅲ.成员方法
  • format方法
    • 将给定的 Date为日期/时间字符串，并将结果追加到给定的 StringBuffer

五、Calendar

1.步骤

  • 看包

    • 需要导包后才能使用

  • 修饰符

    • public
    • abstract
      • 抽象类，不能直接实例化，要用子类

  • 方法概述

  • 构造方法

  • 成员方法

2.构造方法

Ⅰ.无参构造Calendar()

  • 构造具有默认时区和默认 [FORMAT]语言环境的日历。

Ⅱ.带参构造

  • 构造具有指定时区和区域设置的日历。

    • zone - 使用的时区
    • aLocale - 周数据的区域设置

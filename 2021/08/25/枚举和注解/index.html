枚举和注解

发表于 2021-08-25 分类于 Java Valine：
本文字数： 846字     阅读时长 : 1 分钟
了解认识Java中的枚举是什么，怎么使用，以及注解和元注解，还有第三方jar包junit测试的用法

二、枚举

1.概述

  • 发开中，一些固定的值通常用常量存储
    • 常量书写麻烦
    • 不能对其进行其他操作
  • 为了间接的表示一些固定的值，Java就提供了枚举
    • 将固定的数据，一 一 列举出来

2.定义枚举

  • 创建枚举类

    • 枚举中的数据，通常全部用大写（书写习惯）

  • 定义枚举项

3.枚举的特点

  • 所有的枚举都是Enum的子类

  • 我们可以通过 “枚举类名.枚举项名称” 去访问指定的枚举项

    • 枚举项，也是数据本身

  • 每一个枚举项，都是该枚举的一个对象

    • 每个枚举项都会默认走一次构造方法

  • 枚举本质也是一个类，可以定义成员变量

    • 枚举中第一行必须是枚举项

  • 最后一个枚举项后的 ; 可以省略的，但是建议不要省略

  • 枚举类可以有构造方法，但是修饰符必须是private

    • 不写修饰符，默认也是private

  • 枚举类可以有抽象方法，但是所有枚举项必须重写该方法

4.枚举中的方法

Ⅰ. ordinal()
  • 获取枚举项的索引值
Ⅱ. name()
Ⅲ. compareTo(E o)
  • 通常用来比较两个枚举项
  • 返回值是索引值的差值
Ⅳ. toString()
  • 返回对象的内部信息
Ⅴ. valueOf()
  • 获取枚举类中指定名称的枚举
Ⅵ.values()
  • 获取所有的枚举项

三、注解

1.概述

  • @
    • 对我们的程序进行标注和解释
  • 注解和注释的区别：
    • 注释是给人看的
    • 注解是给编译器看的
  • 注解的优点：
    • 更加简洁，方便
  • 注解的作用：就是一个标注的作用

2.自定义注解

  • 格式：

    • 变量的格式

    [^属性类型]: 基本数据类型、String类型、Class类型、注解类型、枚举类型、一维数组+以上类型
    [^default]: 赋一个默认的初始化值

3.练习

  • 自定义一个注解@Test，用于指定类的方法上
    • 如果某一个类的方法上有该注解时，就执行这个方法

四、元注解

1.概念

  • 就是描述注解的注解

  • 即给注解注解的注解

2.三种元注解

Ⅰ.@Target
  • 指定注解在哪使用
  • 默认：ANNOTATION_TYPE 在哪都可以使用
Ⅱ.@Retention
  • 指定该注解存活时间（编译时，运行时）
Ⅲ.@Inherited
  • 表示修饰的自定义注解可以被子类继承

五、junit测试

1.概述

  • 是一个第三方的jar包
    • 可以单独的运行某一个方法

  • Before：在Test之前运行
  • Test：测试这个方法
  • After：在Test之后运行

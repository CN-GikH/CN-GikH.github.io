继承、多态、抽象类和接口

发表于 2021-08-25 更新于 2021-12-11 分类于 JavaSE
本文字数： 1.1k字     阅读时长 : 1 分钟
流程控制--选择结构、循环结构，Scanner键盘录入

一、继承

1.什么是继承？

  • Java中继承

    • 第一步：找到重复代码
    • 第二步：找到封装重复代码的工具
    • 第三步：其他类和公共类没有联系
      • 使用 extends继承

  • 继承的格式

2.继承解决了代码的重复性

开发原则：低耦合，高内聚
  • 耦合：类与类之间的关系
  • 内聚：自己完成某件事的能力

3.继承的构造方法之间的关系

  • 父类与子类的构造方法不会继承

  • 创建子类对象时，会先运行父类的构造方法（初始化夫），然后运行子类的构造方法

    • 因为在继承时，子类构造方法有隐藏的super()

4.继承的成员变量关系

  • 私有的是无法被继承的

    • 实际上是可以继承的，但是不能访问

  • 继承

    • 子类被赋予了访问super内存空间的权限

5.继承的方法关系

  • 方法可以被继承
Ⅰ.方法重写：
  • 概念：子类和父类中的方法声明一模一样，就是方法重写
    • 方法体可以不一样
  • 子类对象调用方法的特点
    • 若方法名不相同，则调用对应名称的方法
    • 若方法名相同，则调用子类自己的
  • 子类重写父类方法时，访问权限不能更低

6.继承的总结

  • Java中只支持单继承，不支持多继承。但是支持多层继承
    • 即 一个类只能有一个爸爸 ，但是可以有一个爷爷，一个祖爷爷
    • 但多层继承打破了封装性，功能唯一性
  • 子类只能继承父类的非私有成员（成员变量、成员方法）
  • 子类不会继承父类的构造方法，但可以通过super()访问

二、多态

1.概念：

  • 某一事物，不同时刻不同状态（扯淡）
  • 赋予父类访问子类的权限
    • 父类要有与子类相同的方法，不然过不了编译
    • 因为对象进实体找方法从上到下

2.结论：

  • 成员变量：
    • 编译看左边，运行看左边 （就近原则）
  • 成员方法：
    • 编译看左边，运行看右边（方法重写）
  • 构造方法：
    • 创建子类对象，先初始化父类
  • 静态方法：
    • 编译看左边，运行看左边（静态和类相关，不是重写）

3.实现多态

  • 要有继承关系
    • 要有方法的重写
  • 父类引用指向子类对象

instanceof

  • instanceof

三、抽象类

abstract

1.什么是抽象类？

  • 为了让父类能够调用子类的方法
    • 而父类必须有一个一样的方法
      • 并且父类方法中不需要写方法体
        • 但不写方法体会报错，需要抽象
  • 一个没有方法体的方法应定义为抽象方法
  • 如果一个类中有抽象方法，则该类必须定义为抽象类

2.定义抽象类格式

  • 抽象类和抽象方法的定义，必须用abstract来修饰

3.抽象类的使用

  • 抽象类无法直接实例化

4.抽象类的组成

  • 构造方法
  • 成员变量
  • 成员方法
    • 是否包含非抽象方法

四、接口

1.接口的定义：

  • 当我们有额外的业务（功能）时，需要用接口区封装业务
  • 接口体现了扩展性

2.接口的格式

  • 接口用关键字interface表示

  • 类实现接口用implements表示

3.接口的组成

  • 构造方法：没有构造方法
    • 不能实例化
  • 变量：没有变量，只能有常量
  • 方法：只能是抽象方法

4.接口的实例化

  • 同抽象方法一样

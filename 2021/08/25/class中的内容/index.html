class中的内容

发表于 2021-08-25 分类于 Java Valine：
本文字数： 1.1k字     阅读时长 : 1 分钟
学习自定义class文件中的成员变量、成员方法、构造方法等

一、自定义的数据类型：引用类型

1.Java中的数据类型分为：

  • 基本数据类型：byte、short、int、long、float、double、char、boolean
    • 整型：byte、short、int、long
    • 浮点型：float、double
    • 字符型：char
    • 布尔型：boolean
  • 引用数据类型：数组、类、接口

2.自定义数据类型（自定义类）的构成

  • 属性：成员变量
    • 成员变量的位置：类中方法外
    • 有默认初始化值（在new时进程进堆内存时初始化）
  • 方法：成员方法
  • 构造方法（构造器）
    • 系统默认给出的
    • 每个类必须要有的

3.成员方法

  • 成员方法的定义

    • 位置：自定义的数据类型（类）中，成员变量后
    • 格式： 修饰符 返回值类型 方法名 （参数列表）{ }
      • 修饰符：public （不用static）

  • 成员方法使用

    • 在main方法中创建自定义数据类型的对象
    • 对象.方法名();

  • 成员方法调用的内存图

4.练习：

  • 狗的数据类型

  • 属性：体重、姓名

  • 行为：

    • 跑
      • 体重-10
    • 吃饭
      • 体重+10
    • 打印
      • 体重和姓名
思路:

  • 创建项目，测试环境

  • 需求分析：

    • 由需求可知，需要定义一个狗的数据类型，Java没有自带这种类型，所以需要自己创建

  • 创建狗的数据类型

    • 位置：class之间并列关系
    • 格式： class 类名 { }

  • 根据需求分析，属性=变量，行为=方法，在创建的数据类型中，编写对应的内部数据

  • 如果想要实现控制台可以重复选择吃饭或跑的功能，需要定义选择，菜单方法

    • 方法一

      • 在main方法并列的地方编写菜单方法

      • 定义选择方法，调用对应的数据

    • 方法二

      • 在自定义的狗类中创建菜单、选择的成员方法

      • 在main主程序中，创建狗类对象，调用狗类选择方法

二、成员变量和成员方法

1.变量和成员变量的区别

Ⅰ.位置
  • 变量：方法内/方法的声明上
  • 成员变量：类中方法外
Ⅱ.初始化值
  • 变量：使用前，需要手动赋值
  • 成员变量：有默认初始化值(在new时进堆内存时初始化)
Ⅲ.同名
  • 变量和成员变量有同名时，系统不会在变量名前提供默认的 this关键字了
  • 就近原则
Ⅳ.生命周期
  • 成员变量：随对象同生共死
  • 变量：方法执行完毕后消失

2.方法和成员方法的区别

  • 成员方法没有static

    • static 静态 ：代码存在与代码区.class文件的静态区中

    • 静态区的内容随着类的加载而存在（栈区可以直接操作静态），优先于对象

      • 栈区不能操作非静态区，但可以直接操作静态区，静态区属于该类本身

    • 静态区的内容属于类，在调用时默认省略的类名

      • 即静态修饰的可以直接通过类名调用

    • 非静态区的内容属于堆区的实例

  • static的注意事项：

    • 在静态方法中没有this关键字
    • 静态方法只能访问静态的成员变量和静态的成员方法

3.类的加载时机

  • 用到类时加载.class进Jvm内存
  • 即碰到new时

三、构造方法（构造器）

1.构造方法的格式

  • 方法名与类名相同
  • 没有返回值，连void都没有
  • 没有具体的返回值

2.什么是构造器

  • 构成类的一个成员
    • 给对象的数据进行初始化
  • 构建实体 : new 类名
  • 构建实体的时候赋值

3.构造方法的细节问题

  • 当构造方法的参数变量名和成员变量名相同时
    • 添加this关键字
  • 构造方法只能赋值，不能改值（在创建对象时完成数据的初始化）

4.补充知识点

  • 如果手动给出构造器，系统不会再自动给出
    • 如果不给，系统给出默认构造器
    • 建议给出空参构造
  • 构造方法可以重载

总结

  • 实体 === 对象
  • 属性 === 成员变量
  • 行为 === 成员方法

集合

一、集合和数组的区别

  • 相同点
    • 都是用来存储数据的
  • 不同点
    • 数组的长度是固定的，集合的长度是可变的
    • 数组可以存基本数据类型，也可以存引用数据类型
    • 集合只能存引用数据类型 int —> interger

二、集合的体系

这个颜色是接口

这个颜色是具体实现类

1.单列集合 Collection

  • 重复的数据 List
    • ArrayList
      • 数组结构，有序的
    • LinkedList
      • 链表结构，无序的
  • 不重复的数据 Set
    • HashSet
    • TreeSet

2.双列集合 Map

  • HashMap
  • Three

三、单列集合 Collection

1.Collection的概念

  • Collection是一个接口，不能直接实例化，需要通过子类实例化

    • 常见的：ArrayList<>

2.Collection的成员方法

① add()方法

  • add(数据) 添加一个数据到集合

  • addAll(集合)，将指定集合中的所有元素添加到此集合

② clear()方法

  • 从此集合中删除所有元素（可选操作）

    ③ size()
    ④ remove()

3.集合的遍历

因为collection类中没有定义get方法，所以不能通过get取数据

  • 遍历集合的第一种方法，用增强for

    • ```java

      for (int i : arr){
    System.out.println(i);
}
      • iterator中
        • 有一个next()方法，按顺序返回下一个元素
        • hasNetx()方法
          • 如果还有元素，返回true
          • 如果没有元素，返回false

    • 使用iterator遍历数组

四、单列集合中的 List

1.List的概述

  • 接口
  • 有序的（怎么放的，怎么拿）

2.List的成员方法

  • 拥有get()方法，可以对索引进行操作
  • List集合也可以使用迭代器
  • add（）、clear（）、equals（）、remove()、等方法

3.List的特点

  • 存取有序
  • 可以重复
  • 有索引

4.List特有(常用)的方法

  • ```java

    • void add (int index,E element) 添加
    • remove(int index) 根据索引删除
    • set(int index,E Element) 修改
    • get(int index) 获取所在索引值

  • Set集合是无序的

二、HashSet

  • 允许存null值
  • 对迭代顺序不做任何保证
  • 容量和size是两个概念
    • 容量：容器的长度
    • size：内容的长度
  • 开发中ArrayList集合使用的较多
  • HashSet没有索引
  • HashSet底层数据结构是：哈希表+链表结构

1.构造方法

Ⅰ.无参构造
  • 实例具有默认初始容量（16）和负载因子（0.75）。
Ⅱ.带参构造

  • 参数传递一个Int类型的数据，是集合指定初始容量

Ⅲ.带参构造

  • 参数传递一个Collection集合，重复的值不会录入

2.成员方法

  • 方法都有返回值，可以通过接收add、remove等方法的返回值（布尔类型），来判断是否操作成功
Ⅰ.add()方法

  • 将指定的元素添加到此集合（如果尚未存在）

Ⅱ.remove()

  • 如果存在，则从该集合中删除指定的元素

    • HashSet没有索引
Ⅲ.clear()
  • 删除集合中的所有元素
Ⅳ.size()
  • 返回集合中的元素数
Ⅴ.iterator()
  • 返回集合中元素的迭代器

三、TreeSet

  • Set是无序的
  • TreeSet可以自定义排序顺序

1.构造方法

Ⅰ.无参构造

  • 构造一个新的，空的树组，根据其元素的自然排序进行排序

    • 自然排序：自动默认排序，根据哈希码或ASCII码
Ⅱ.有参构造(集合)

  • 构造一个包含指定集合中的元素的新树集，根据其元素的 自然排序进行排序

    • 带参一个集合
Ⅲ.带参构造(Comparator自定义顺序)
  • 构造一个新的，空的树集，根据指定的比较器进行排序。
  • 先欠着，以后晚自习将

四、泛型

1.什么是泛型

  • <>中的东西

  • 泛型实际的含义就是任意类型（Object）

    • jdk5以后引入的新特性，为了提高编译时的类型安全检测机制

  • Java中出现 E V K ：指的就是任意类型

2.泛型的优点

  • 可以将运行时的异常提前到编译时

  • 避免了数据类型的强制转换

  • Java中如果没有明确标明集合类型，则集合中的数据不管是什么类型，都会向上转型为Object类型

3.定义泛型类

  • 定义格式

  • 定义泛型方法

  • 定义泛型接口

  • 定义泛型类、泛型方法的意义：告诉使用者，我这个方法可以传任意的数据

1.<?>

  • 通常用于方法定义参数，不知道传递什么类型时，使用<?>

2.类型的通配符上限<? extends 类型>

3.类型通配符的下限<? super 类型>

六、Map(双列集合)

  • 是一个泛型接口
  • 该容器，能够存储两列
    • 1
    • 键值对关系，通过键去取值
  • 将键映射到值的对象。 双列集合不能包含重复的键，每个键可以映射到最多一个值。
    • 键：Key
    • 值：value

1.存数

  • 使用.put()方法

    • 将指定的值与该映射中的指定键相关联（可选操作）
    • key键值不能重复
    • value可以重复

  • 如果存入相同的键，则随机抛出一个

  • put()方法返回值是返回 之前覆盖掉的数据

2.取值

Ⅰ.可以使用get()方法取值

  • 但是键值不是固定的顺序

  • 不能通过for循环遍历集合中的键值对

Ⅱ.entrySet
  • 是Map的内部类
    • 为了保存K和V
      • 但键和值并不是个别的，有很多个
    • 所以将得到的值封装到一个Set集合中
  • 返回值为Map.Entry类型（内部类）
  • 返回A双列集合中包含的映射的键值对。该集合由A双列集合支持，因此对双列集合的更改将反映在集合A中，反之亦然。
  • 即定义一个Set集合，保存entrySet()方法返回的数据
for循环+Map.Entry取值
Iterator迭代器+Map.Entry取值
Ⅳ.keySet()方法取值

  • 返回此双列集合中包含的键的[Set]视图。

    • 取出map集合中的键，并存入Set集合
for循环+keySet()取值
Iterator迭代器+keySet()取值

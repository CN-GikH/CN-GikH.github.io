学习网络编程，TCP、UDP协议传输数据的方法及区别

一、网络编程

1.网络编程的概念

  • 学习Java的操作
    • 操作内存
    • 本地存储（IO流）
    • 和互联网中的设备产生关系

2.网编的三要素

  • 网络协议（网络通道）
  • IP(设备的地址)
  • 端口号（具体软件）
    • 0-65535

二、InetAddress

1.API概述

  • 包java.net.InetAddress
    • 导包
  • 修饰符public
  • 使用InetAddress
    • 构造方法
      • 没有构造方法
    • 使用静态方法
      • 通过类名直接调用

2.成员方法

Ⅰ. getByName()
  • 确定主机名称和IP地址
  • 能获取自己的，也能获取别人的
Ⅱ. getLocalHost()
  • 返回本机的地址
    • 只能查看自己的，本机的IP
Ⅲ. getHostName()
  • 获取此IP地址的主机名
  • 返回值String类型
  • 没有static修饰
Ⅳ. getHostAddress()
  • 获取IP
  • 返回值String类型
  • 没有static修饰

三、网络协议

1. TCP/IP

  • 应用层
    • http ftp DNS
      • 应用软件之间的通信 — > 前端开发
  • 传输层
    • TCP UDP
      • 负责网络程序通信 — > 后台开发
  • 网络层
    • ip icmp IGMP
      • 负责数据传输的目标
  • 链路层
    • 驱动程序、接口
      • 硬件部分

2.传输层UDP (快递)

  • 面向无连接通信
    • 发送端和接收端不建立逻辑关系
    • 不在乎对方是否存在/是否接收
    • 耗费资源比较小，通信效率高，通常被用来音频、视频、普通数据的传输

六、UDP传输接收端

DatagramSocket：可以用来发送也可以用来接收

1.构造方法

Ⅰ. DatagramSocket(int port)
  • 创建该类实例，并指定要接收的端口号

2.成员方法

Ⅰ. receive(DatagramPacket p)
  • 参数需要传递一个DatagramPacket对象
    • 即创建一个报包，用于接收数据

总结：UDP 发送接收数据报包的步骤

一、发送

  • 将要发送的数据存入byte[]数组

  • 将数据打包

    • 使用DatagramPacket的构造方法将数据打包

    • 指定byte[]数据，长度，给出InetAddress对象(地址)，端口号

  • 准备发送的工具（快递小哥）

    • 使用DatagtamSocket构建一个对象，调用它的send方法，发送数据

    • send方法需要传递一个DatagramPacket类的对象

二、接收

  • 首先要准备一个DatagramPacket对象，用于接收发送端发送过来的数据

    • 使用简单的构造方法，构造一个DatagramPacket，需要一个byte[]数组

  • 创建接收数据报包的工具

    • 使用DatagramSocket构建一个接收数据报包的工具，带参传递一个端口号

      • 端口号需要和发送端一致，才能接收到数据

  • 接收数据

    • 调用DatagramSocket的receive方法，接收数据

      • 需要传递一个DatagramPacket对象，用于存储接收的数据
        • 使用之前准备好的dp，接收数据

  • 将接收到的，存在报包中的数据，取出

    • 通过.getLength()方法获取数据报包的长度，因为默认编码不支持中文
      • 如果需要中文可以进行转换为String类型
    • 通过.getData()方法获取数据包中的数据
      • 返回值是一个byte[]数组类型，用一个数据进行存储
    • 遍历数组进行查看数据

  • 严格区分客户端和服务器端

七、TCP服务器端ServerSocket类

1.概述

  • 包：java.net.ServerSocket
    • 需要导包
  • 修饰符：public 直接使用
  • 属于网络流，一般服务器端不关流

2.构造方法

Ⅰ. ServerSocket(int port)

Ⅱ.其他构造

3.成员方法

Ⅰ. accept()

  • Socket：返回值类型

  • 等待 侦听/接收 客户端数据，没有就一直等待

Ⅱ. getInputStream()
  • 以客户端为参照物，服务器端用字节输入流，保存数据

八、TCP客户端Socket

1.概述

  • 包：java.net.Socket

    • 需要导包

  • 修饰符：public 直接使用

  • 属于网络流，用完需要关流

2.构造方法

Ⅰ. Socket(InetAddress a, int p)

3.成员方法

  • 客户端向服务器端发送数据
    • 实际上依然是通过IO流进行读写
  • 又因为服务器端不能主动请求和发送数据（例如NVC）
    • 所以IO流要以客户端为参照物
Ⅰ. getOutputStream()
  • 以客户端为参照物，用字节输出流写入数据到服务器端

服务器端回复消息到客户端

  • 步骤

    • 以服务器端为中心

    • 向客户端用字节输出流写入数据

    • 客户端用字节输入流接收数据

    • 客户端遍历接收到的数据并打印到控制台

  • 注意事项：如果客户端要接收服务器端的返回数据

    • 必须先将客户端之前向服务器端写入的输出流关闭
      • 否则服务器端会一直处于等待数据状态，不会继续执行操作
      • 也就是read处于阻塞状态
    • 通过关闭客户端输出流，解决阻塞状态

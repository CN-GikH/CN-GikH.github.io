一、医疗项目前端

1.前端常用技术

  • ES6， 全称 ECMAScript 6.0 ，是 JavaScript 的下一个版本标准，2015.06 发版
    • ES6 主要是为了解决 ES5 的先天不足，比如 JavaScript 里并没有类的概念，但是目前浏览器的 JavaScript 是 ES5 版本，大多数高版本的浏览器也支持 ES6，不过只实现了 ES6 的部分特性和功能
  • vue.js也是前端的JS框架为了简化WEB开发
  • axios 是一个独立的vue项目，可以用于浏览器和node.js中发送ajax请求
  • Element-ui 网站快速成型工具
  • Node.js

二、Node.js

1.概念

  • 是一个基于 Chrome V8 引擎的 JavaScript 运行环境
  • 它分为两个部分
    • DOM渲染引擎
    • JavaScript解析引擎

2.什么是Node.js？

  • 可以脱离浏览器运行javascript，需要有javascript引擎

3.安装Node.js

  • 官网下载，无脑点点点，安装即可

    • cmd 运行 code -v 查看版本

    • 同时还需要查看依赖npm -v的版本

  • 设置npm镜像为淘宝镜像

  • 安装vue的脚手架

    • 使用vue -V查看脚手架是否正确安装

4.医疗前端项目环境准备

  • 将前端项目解压，并用HBuilder打开

    • 在项目文件路径下进入cmd

    • 我们需要安装node-sass（需要在项目路径下进行安装）

    • 再次run dev项目，可以看见项目就能跑起来了

5.实现前后端交互

  • 现在前端项目以及跑起来了，为了让它能够和我们的后端实现连接交互，我们还需要对前端项目进行一些配置

  • 找到前端的config文件夹中的dev.env.js

    • 对BASE_API的路径进行配置，添加端口号

    • 对index.js中的proxyTable（代理表格）进行配置跨域

    • 在src/api/hosp.js等js文件中可以看见前后端请求交互传输的地址

三、医疗项目实现前后端交互

1.解决跨域问题

  • Controller中添加@CrossOrigin跨域注解

2.添加分页插件

  • 设置前端、后端跨域，设置mapping映射路径一致

启动项目进行测试，可以查询到数据

测试医院设置列表中的功能是否正常运行

3.实现数据字典

  • 什么是数据字典？

    • 是管理系统常用的分类或者一些固定的数据
      • 比如省市区三级联动、民族、行业数据、学历数据

  • yygh_cmn数据库作为数据字典存储数据

    • 字段有Id ，父级ID ，name ,value ，创建时间 ，修改时间 ，是否删除
    • 插入数据

  • 在service下创建service-cmn项目

    • 复制一份properties配置文件过来进行修改
    • 修改项目名称，端口号，数据库连接

    • 创建启动类

    • 创建mapper.DictMapper类，操作数据库（MyRatePlan封装）

      • 继承BaseMapper
      • 添加Dict泛型
      • 添加@Mapper注解

      • 同包下创建对应的DictMapper.xml配置文件，引入约束头，以后有需要添加的业务可以进行配置
      • 创建根据ID查找子列表的方法

    • 创建service.DictService业务逻辑层接口

      • 继承Iservice
      • 添加Dict泛型
      • 创建DictService的实现类
        • 添加@Service注解
        • 继承ServiceImpl
          • 添加泛型DictMapper,Dict
      • 重写父类根据ID查找子列表方法

      • 创建判断ID是否有子节点的isChilder方法

    • 创建controller.DictController控制器

      • 注入service对象
      • 添加@RestController注解
      • 添加@RequestMapper映射路径
        • 打开前端api/dict.js查看前端访问的路径进行填写

    启动项目进行测试，查看数据管理中的数据字典，可以查询到数据

4.easyexcel实现数据字典导入导出

easyexcel概念
  • EasyExcel是阿里巴巴开源的一个excel处理框架
  • 使用简单，节省内存，
  • EasyExcel能大大减少占用内存的主要原因是在解析Excel时没有将文件数据一次性全部加载到内存中，而是从磁盘上一行行读取数据，逐个解析。
    • Apache poi,jxl等，都存在一个严重的问题就是非常的耗内存，如果你的系统并发量不大的话可能还行，但是一旦并发上来后一定会OOM或者JVM频繁的full gc
easyexcel使用

  • 第一步当然是我们的添加依赖咯

  • 使用DictEeVo实体类，在Service中添加exportData导出数据的方法

    • 实现类中重写exportData方法，进行一些列的response响应设置

      • 表格拓展名改为xls

    • 在controller控制器中添加点击导出时访问的方法

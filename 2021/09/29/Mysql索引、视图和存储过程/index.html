一、Mysql索引

1.概念

  • Mysql的索引和数组的索引?
    • 是一样的
    • Mysql索引是加在字段上的
    • 目的：提高了查询效率

2.Mysql索引的分类

索引名称 含义
主键索引（primary key） 和主键的含义一样；查询速度快
唯一索引（unique） 索引值是唯一的
普通索引（index） 最常见的索引；加快对数据的访问速度

3.主键索引

  • 特点：唯一，非空，每个表只能有一个主键索引（用来标识每一条数据）

  • 添加主键索引

    • 可以在建表的时候添加主键索引
    • 创建表之后，修改表结构添加索引

4.唯一索引

  • 特点：所有的值只能出现一次，必须唯一

  • 加上唯一索引，查询的速度会提高

    • 在开发中，通常利用的是唯一索引的约束作用（唯一性）

  • 添加唯一索引

    • 创建表时添加唯一索引

    • 使用CREATE语句创建（在已有表上创建索引）
    • 使用ALTER语句创建索引

5.普通索引

  • 目的：有些字段查询、分组使用频率比较多，建议该字段作为普通索引

  • 添加普通索引

    • CREATE方式
    • ALTER方式

6.删除索引

  • 格式

7.测试索引性能

  • 如果数据量较大，在创建表时应该添加索引，否则有可能失败

  • 添加索引后，速度更快

索引的优点和缺点

添加索引首先应考虑在where和order by 涉及的列上做添加

  • 优点
    • 提高了查询的速度
    • 显著的减少查询中分组和排序的时间
  • 缺点
    • 创建索引和维护索引需要时间，而且数据量越大时间越长
    • 当对表中的数据进行增加、修改、删除的时候，索引也会同时进行维护的，降低了数据的维护速度

二、Mysql视图

1.概念

  • 什么是视图？
    • 也是数据库的一部分
    • 视图是一张虚拟表
    • 视图建立在已有表的基础上，这些原基础的视图被称为**基表**
    • 视图提供数据内容语句为SELECT语句，可以将视图理解为存储起来的select语句
    • 视图向用户提高基表数据的另一种表示方式
  • 视图的作用
    • 权限控制的时候可以使用
      • 视图操作不会影响到原表
    • 简化复杂的多表查询

2.创建视图

  • 将Select语句查询结果作为基表
    • 字段名可以不写，如果不写会自动使用基表字段名

3.查询视图

  • 和查询表的操作一样

4.视图和表的区别

  • 视图是建立在表之上的，表存储的是数据库中的数据，而视图存的是一个数据的展示(select 查询语句)
  • 通过视图不能改变表中的数据（一般情况下视图中的数据都是表中的列，经过计算得到的结果，不允许更新）
  • 删除视图，表不受影响
    • 但是删除表，视图不再起作用

三、存储过程

1.概念

Mysql5以后开始支持存储过程

  • 什么是存储过程？

    • 存储过程是一种在数据库中存储复杂程序，以便外部程序调用的一种数据库对象
    • 存储过程是为了完成特定功能的SQL语句集
    • 简答来说：存储过程就是将一堆SQL语句的合并，中间加入一些逻辑控制
    • 类似于Java中的方法

  • 存储过程的优缺点

2.创建方式

  • 数据准备

    • 创建一个商品表和一个订单表

  • 创建存储过程

    • 方式一

      $$ ： 声明语句的结束符号，可以自定义，一般使用

    • 方式二

      • 创建存储过程，可以携带参数

    • 方式三

      • 调用存储过程时，可以定义一个返回值

      • 定义变量

      • 返回值

        • 返回变量值使用 SELECT @变量名 即可
        • 调用存储过程时，因为返回值定义在声明上，所以调用要写上变量名

  • 总结
    • 定义变量一定要加SET和@符号
      • SET @变量名 = 数值
    • 如果有输入，需要加 IN
    • 如果要返回值 ，需要 OUT
    • 带回返回值 SELECT

四、触发器

1.概念

  • 什么是触发器

    • 当我们执行一条SQL语句时，这条SQL语句的执行会自动的触发其他的SQL语句

    • 类似于Java中的监听器

2.触发器四要素

  • 监视的地点
    • 监视的哪张TABLE
  • 监视事件
    • insert / update / delete
  • 触发时间
    • before / after
  • 触发的事件
    • insert / update / delete

3.定义触发器的语法格式

$ 定义触发器的结束符号

  • 定义

    • 在同一个数据库中，触发器的名称是唯一的，不能重复

六、DCL操作（控制语言）

1.概念

  • Mysql默认使用的是root账户，超级管理员，拥有全部的权限
    • 使用DCL可以创建不同的用户并赋予不同的权限

2.创建用户

  • 主机名 = IP地址
参数 含义
用户名 创建的新用户的登录名称
主机名 指定该用户可以在哪个主机上可以登录，本机登录的可以使用localhost，如果想该用户远程登陆的话，主机名就写通配符 %
密码 登录密码

3.DCL分配权限

  • 用户授权

    • 操作有插入，修改，删除等，每一种都需要给予权限
    • 分配所有权限

  • 查看用户权限

    • ```sql
      show grant for ‘用户名‘@’主机名’

  • 查看所有用户

    • SELECT * FROM USER

数据库知识

发表于 2021-09-17 分类于 数据库
本文字数： 2k字     阅读时长 : 3 分钟
学习Mysql数据库的基本知识，安装配置环境，学习SQL语句,DDL,DML,DQL

一、数据库

1.概念

  • 对数据进行存储和管理的一个仓库

  • 本质还是一个文件系统，以文件的形式保存在电脑上

  • 数据库真正的存储在C盘ProgramData的Mysql文件夹下

2.为什么使用数据库

  • 存储数据的方式

    存储方式 优点 缺点
    内存 速度快 临时
    文件 数据可以永久存储 IO流，不方便
    数据库 永久存储、方便管理和存储数据、统一的方式操作数据库(sql) 占用资源，收费(Mysql，Oracle)

二、Mysql数据库的相关操作

1.Mysql的卸载

  • 删除C盘ProgramData中的Mysql文件夹即可，不需要删除注册表

2.Mysql的安装

  • 第三种第三种
    • Client only 客户端
    • Full 所有

  • 如果电脑用户名中文，勾选show….

  • 如果用户名中文，会出现乱码，在C盘创建一个.log文件，选中即可

3.配置Mysql环境变量

需要注意的是，Mysql安装完成后会出现两个目录

  • 安装目录：可执行程序
  • 文件目录：保存sql文件

  • 找到Path环境变量，添加路径

4.Mysql的启动和关闭

  • 方式一
    • win+r打开运行，输入services.msc服务管理
    • 打开Mysql服务，启动即可
  • 方式二（管理员命令行）
    • net start 服务的名称
    • net stop 服务的名称

5.Mysql的登录

mysql -u用户名 -p密码 使用指定账户和密码进行登录（root）
mysql -h主机ip -u用户名 -p密码 -h 指定ip（远程登录）
  • Mysql的图形化管理界面
    • sqlyog

6.Mysql的目录结构

  • Mysql的安装目录（C:\Program Files\MySQL）

    • 目录 作用
      bin 可执行文件
      docs 文档
      include 头文件
      lib 依赖
      share 用于存放字符集、语言等信息
  • data目录：C:\ProgramData\MySQL\MySQL Server 5.7
    • data:存放数据的（文件）
    • my.ini :mysql的配置文件（建议不要更改配置文件）
  • 总结
    • 数据库：文件夹
    • 表：文件
    • 数据：文内内容

三、数据库管理系统

1.概念

  • Database ManageMent System ==> DBMS
    • 指一种操作维护和管理数据库的一个大型软件
  • Mysql就是一个数据库管理系统软件，安装了Mysql软件的电脑，这台电脑就被称为数据库服务器

2.数据库管理系统的作用

  • 用于建立、使用、和维护数据库，对数据进行统一的管理

3.数据库管理系统、数据库、表之间的关系

  • 数据库管理系统：数据库软件
  • 数据库：一个应用
  • 表：不同的服务

四、表

  • 数据库的基本单位就是表

  • 表和Java的关系

    • 表的基本单位是：列
      • 列名
      • 数据类型
    • Java的基本单位：类（对象）–> 用成员变量存储数据
      • 变量名
      • 数据类型

    表 ——— 类

    行（属性） ——- 一个对象

    列 ———– 成员变量

五、sql

1.概念

  • 什么是SQL？
    • Structrued query Language ：结构化查询语言，简称SQL
  • SQL的作用
    • 所有的关系型数据库统一的操作规范
    • 不同的关系型数据库都支持sql语句
    • 可以看作一种特殊的编程语言
    • 用来操作数据库

2.SQL通用语法

  • sql语句可以写多行，也可以写单行，以 ; 结尾
  • 可以使用空格和缩进来增加语句的可读性
  • 不区分大小写，一般关键字是大写
    • 数据库名，表名，列名 一般小写

3.SQL中的注释

注释语法 作用
– 空格 单行注释
/**/ 多行注释
# Mysql独有的注释方式（方言）

4.SQL语言的分类

分类 作用
数据的定义语句（DDL） 用来定义数据库、表和列
数据操作语句（DML） 用来对数据库中的数据进行 增删改 操作
数据查询语句（DQL） 用来对数据库的数据进行查询
数据的控制语言（DCL） 用来访问的权限和安全级别

六.DDL操作数据库

1.创建作数据库

  • 创建数据库

指定字符集，如果指定UTF-8，只需要写u即可

2.查看和选择数据库

命令 作用
use 数据库名 切换数据库
select database() 查看当前正在使用的数据库
show databases 查看所有的数据库(DBMS系统中所有的)
show create database 数据库名 查看数据库的定义信息

3.修改数据库

命令 作用
alter database 数据库名称 character set 字符集 修改数据库字符集

4.删除数据库

  • drop database 数据库名

七、数据类型

1.常用数据类型

数据类型 作用
int 整型
double 浮点
varchar 字符串（可变）
data 日期（只要年月日，没有时分秒）

2.varchar和char的区别

  • char：固定的
    • 相当于Java中的数组
      • 使用char时也可以（）指定长度，但是一旦指定就是固定的
    • 存账户密码时用的较多
  • varchar：可变的
    • 相当于Java中的集合

八、DDL操作表

1.创建表

  • 当我们需要复制一个表时，可以使用like关键字

2.查看表

命令 作用
show tables 查看所有的表
desc 表名 查看表结构

3.删除表

  • 直接删除
  • 先判断存不存在，再删除

4.修改表

  • 修改表名
  • 修改字符集
  • 添加列
  • 修改列名
  • 修改数据类型和长度
  • 删除字段

九、DML操作（SQL语句的更新操作）

1.添加数据

  • 第一种

    • 如果时字符串类型，需要用 ‘ ‘ 括起来

  • 第二种

    • 可以不写字段，但是每一个数值需要对应相应的字段

  • 第三种

    • 指定列添加

  • 注意

    • 给所有的字段添加时使用VALUES
    • 字段名称和要添加的数据一一对应
    • 字符串必须使用单引号和双引号包裹
    • 不给字段添加数据时，可以不写，也可以写NULL

2.修改数据

  • 更改指定列的所有值
  • 指定条件修改
  • 修改多个字段
    • 多个 字段名=数值 之间加 ，

3.删除数据

  • 删除指定数据
  • 删除所有（不推荐，效率低）
  • 删除所有（推荐，效率高）
    • 先删除整张表，再新建一个一模一样的表

十、DQL（查询）

1.简单查询

  • 显示所有的字段
  • 只显示指定字段
  • 起别名AS的用法
    • 表和字段都可以起别名

2.其他查询

  • 去重
    • 不显示重复数据
  • 条件查询
运算符 作用
> , < , = , >= ,< = , != 大于小于….
between x and x 在.x…x之间
IN（集合） 集合表示多个值 in中的每一个数据都会作为一次条件，只要满足该条件就会显示
like ‘%网%’ 模糊查询
is null 查询某一个列上为 null 的值

3.模糊查询

4.逻辑运算符

逻辑运算符 作用
AND & 与
OR || 或
NOT 取反

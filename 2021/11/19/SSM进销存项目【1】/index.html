SSM进销存项目和Eclipse基础配置

发表于 2021-11-19 更新于 2021-12-11 分类于 SSM进销存项目
本文字数： 890字     阅读时长 : 2 分钟
使用Eclipse工具，使用SSM框架，服务器：Tomcat;整合搭建经销存项目 --- 第一部分（包含eclipse的基本配置）

今日内容

  • SSM复习

一、Eclipse的配置

1.安装

  • 下载eclipse安装包，解压到要安装的目录
  • 直接双击eclipse.exe即可启动
    • 跳出弹窗选择工作空间

启动安装eclipse之前，需要配置好jdk的环境变量

2. JDK配置

  • 默认JDK配置
    • window ==> Preferences ==> Compiler
    • window ==> Preferences ==> Installed JREs

3.自动补全提示

  • 依次点击Window –> Perferences（选项设置） –> Java –> Editor（编辑） –> Content Assist
  • 然后在“Content Assist”页面中，可以看到下面有一个Auto Activation中有一个Auto activation triggers for Java
  • 在这一个输入框中输入“.qwertyuioplkjhgfdsazxcvbnm”
    • 即所有案件都会触发自动补全

4.Tomcat配置

  • 点击Window –> Perferences（选项设置）–> Server –> Runtime Environments

  • 点击Add添加Tomcat服务器

  • 选择Tomcat版本

5.eclipse主题配置

  • 打开eclipse——>help——>Eclipse Marketplace，在search搜索框里输入：DevStyle

  • 安装DevSeyle主题

  • 根据提示重启idea，选择相应的主题颜色

  • 点击Window –> Perferences –> DevStyle可以对DevStyle主题进行自定义配置

6.eclipse设置自动保存

  • windows-> Preferences -> General -> Workspace
    • 勾选 “Save automatically before build“
    • 修改时间为1
  • windows->Preferences -> Run/Debug -> Launching -> Save dirty editors before launching 设置为 Always

7.eclipse导入项目中文乱码问题解决

  • 右键项目名，选择Properties属性，选择Resource

    • 修改编码格式为GBK或UTF-8，反复尝试几次

    如果没有GBK或UTF-8的选项，则手动打上去即可

  • 如果上述操作后还是乱码，是java source file 编译格式不一致

    • 找到Window–>preference—->general—–>content type————>Text——–java source file
    • 修改GBK或UTF-8

8.eclipse设置包名分层级显示

  • 设置完成后，还是和没设置相比有一些分层的，但是不像idea每一个包名分层，后面再研究

9.eclipse配置maven环境

  • 找到找到Window–>preference–>Maven–>Installations

    • 点击Add添加maven
    • 找到maven的安装目录即可

  • 同级目录找到User Setting ，配置setting.xml 配置文件的路径

10.彻底卸载eclipse

  • eclipse不像idea一样需要安装，可以说是免安装版的，如果要卸载只需找到eclipse的解压目录，删除整个eclipse文件即可
  • 如果想要卸载干净，同时找到c盘下的这些文件夹进行删除
    • .eclipse
    • .p2
    • workspace

二、SSM项目

1.Supplier供应商页面显示

  • 创建SupplierController层

    • 根据网页路径，定义供应商后端程序名称

    • 创建list()方法，用于返回list页面

  • 在WEB-INF的views视图解析器中创建supplier文件夹

    • 创建list.jsp页面

  • 在entity.admin中新建Supplier供应商Bean类

    • 添加@Component注解

    • 添加get/set/toString方法
      • 快捷键alt+shift+s

  • 在dao.admin中创建SupplierDao层

    • 添加@Repository

  • 在config/mybatis/mapper/admin中创建SupplierMapper.xml文件

    • 复制约束
    • 修改namespaces为SupplierDao

  • 在server.admin中创建SupplierService接口

    • 复制Dao层中的方法即可

    • 创建SupplierServiceImpl，实现接口
    • 添加@Service注解
    • 重写方法
    • 注入supplierDao层接口，每个方法调用相应的Dao层方法即可

  • 修改list.jsp中的内容

2.Supplier供应商添加数据

  • 根据前端页面也可以查看当用户点击添加按钮时触发的函数

  • 在Supplier的Controller层创建添加的方法

    • 注入SupplierService对象噢

3.Supplier供应商显示数据

  • 在Supplier的Controller层创建显示数据的方法

4.Supplier供应商编辑数据

  • 在Supplier的Controller层创建编辑数据的edit方法

    • 注意注解@ResponseBody

5.Supplier供应商删除数据

  • 在Supplier的Controller层创建删除数据的edit方法

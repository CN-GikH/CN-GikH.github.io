一、Redis

2.关系型数据库和非关系型数据库

  • 关系型数据库
    • MySQL、Oracle、SqlServer、SqlLite、PostgreSQL（大象数据库）
    • 关系型数据库的优点
      • 易于维护：用表存数据，格式是一致的
      • 使用方便：sql-92，SQL语法大部分是通用的
      • 复杂操作：多表查询
    • 关系型数据库的缺点：
      • 读写性能差：海量数据来的时候
      • 数据库表结构固定，灵活性差一些
      • 高并发的需求硬盘的I/O瓶颈
  • 非关系型数据库
    • Redis就是非关系型数据库的典型代表、MongoDB
    • 非关系型数据库严格来讲不是数据库，而是数据存储结构化的集合，使用key/value存储
    • 非关系型数据库的优点
      • 格式灵活，以 key / value 方式存储（文本、文档、图片）
      • 速度快
      • 成本低 nosql部署简单，所有的软件 均是开源的
    • 非关系型数据库的缺点
      • 不提供sql支持
      • 无事务支持
      • 结构有点复杂

3.什么是Redis

远程字典服务，支持网络，可基于内存亦可持久化，Key-Value数据库

  • 支持存储的value类型很多，包括string、list、set、zset(有序集合)、hash(哈希类型)
Ⅰ. String类型

  • 在redis中String类型比较多，而且是二进制比较安全的

    • Map<String,String> map

  • redis中的基础命令格式：

    • 类型命令 key 参数数据

    • 存入键值对

    • 根据键取出值

    • 把值递增1

    • 把值递减1

    • 删除键值对

    • 设置有效时间

      • timeout：失效时间，单位s
Ⅱ. Hash类型

  • 将一个对象存储到hash中，会比存储到String占用更少的空间

    • Map<String,Map<String,?>> map

  • 格式

    • 存入一个Hash对象
    • 根据hash对象键取出值
    • 判断hash对象是否含有某个键
    • 根据hashkey删除hash对象键值对
Ⅲ. List类型

  • 链表结构数据类型，主要功能push pop 获取元素

    • Map<String,List>
      • right = r

  • 格式

    • 往列表右边添加数据
    • 往列表左边添加数据
    • 弹出列表最左边的数据
    • 弹出列表最右边的数据
    • 显示列表数据
      • 查询全部 0 -1
    • 获取列表数据

二、SpringBoot装配原理

  • starter场景启动器
    • 我们一直在用官方或第三方提供的starter
    • 想要自定义一个starter启动器，需要先理解SpringBoot的装配原理
  • SpringBoot和SpringMVC的区别
    • 使用SpringMVC时需要配置的信息
      • DateSource
      • 包扫描
      • mvc配置
      • 事务配置
      • … ….
    • 使用SpringBoot时
      • 不用配置，只需提供starter场景启动器即可
      • 自动装配

  • 自动装配，以Mybatis为例

    • 这里即启动了自动装配，可以看到自动装配类是MybatisAutoConfiguration

找到Mybatis的自动装配类进行学习

  • @Configuration

    • 表示的是配置类、配合@Bean帮我们创建对象

  • @ConditionalOnClass

  • @ConditionalOnMissingBean

    • 配合@Bean标签，当没有sqlSessionFactory时创建一个对象

  • @ConditionalOnBean

    • 在Spring容器中需要有DataSource对象，本配置类才能生效
    • 有一票否决制

  • @EnableConfigurationProperties

    • 加载括号中的属性配置类

  • @AutoConfigureAfer

    • 此配置类加载时机晚一些
